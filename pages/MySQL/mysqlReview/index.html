# 数据库技术与应用复习大纲

1. 数据库 (DB)、数据库系统 (DBS)、数据库管理系统 (DBMS) 关系： 数据库系统包含数据库 、 数据库管理系统
2. 常见关系型数据库： Oracle 、 MySQL 、 SQL Server
关系数据库的层次结构： 数据库 、 表 、 记录 、 字段（列）
3. 在 windows 命令行中：启动 mysql 服务： net start mysql服务名 ；停止 mysql 服务： net stop mysql服务名
4. 数据库操作： create database 数据库名 、 show databases 、 drop database 数据库名
5. 主要概念：
1）. 主键：主键列值唯一、不空
unique 约束列自动创建 unique 索引，可空，表中无主键时第一个升级为聚集索引
2）.MySQL 中变量： 系统变量 、 用户变量 、 局部变量
3）. 数据模型三要素： 数据结构 、 数据操作 、 数据完整性约束
4）. 关系模型用二维表表示实体与实体之间关系
5）. 视图：虚拟表、依赖于基本表或视图
6）.path 环境变量：在任何位置使用 mysql 命令
7）. 索引：索引（Index）是帮助 MySQL 高效获取数据的数据结构（有序），在数据之外，数据库系统还维护着满足特定查找算法的数据结构，这些数据结构以某种方式引用（指向）数据，这样就可以在这些数据结构上实现高级查找算法，这种数据结构就是索引。索引是存储在磁盘上的 数据结构 、包含对数据表中所有记录的 引用指针 ，目的是为了提高 查询效率
视图可以使使用更加简单、保护数据安全、保证数据独立作用
8）.mysql 中 外键的取值是另一表（父表）的主键值，也可为空；一个表可以有多个外键，通过外键两个表建立连接
9）.check 约束：设置字段值的取值规则
10）.MySQL 日志：错误日志、通用日志、二进制日志、慢查询日志 (填空)
11）. 数据库的特点：实现数据独立性、实现了数据共享、减少了数据冗余度、实现了数据集中控制、维护了数据完整性 (简答)
12）.SQL 语句分为： ddl（数据定义语言） 、 dml（数据操纵语言） 、 dql（数据查询语言） 、 dcl（数据控制语言）(简答)
6. 常见系统函数：
日期时间函数： now 、 curtime 、 curdate 等
字符串函数： repeat 、 length 、 concat(字符串连接) 、 upper 、 lower 、 substring 等
数值函数： ceil 、 floor 、 mod 、 round 等
流程控制函数： ifnull ， if 等
JSON 函数： json_array_append 、 json_remove 等
set @m=‘[“a”,[“b”,“c”],“d”]’,@n=‘{“a”:1,“b”:[2,3]}’
select json_array_append(@m,‘[1]′,2),jsonremove(@n,′[1]',2),json_remove(@n,'[1]′,2),jsonr​emove(@n,′.a’)
7. 游标的声明: declare 游标名称 cursor for 查询语句
索引的创建: create [unique|fulltext] index 索引名 on 表名(索引列列表)
存储过程的创建： create procedure 过程名(参数) begin ... end
存储函数的创建： create function 函数名(参数) returns 返回值类型 characteristic begin ... return 值 end
调用存储过程和函数 call 过程名、select 函数名
触发器的创建：create trigger 触发器名称 before/ after insert /update/delete on 表名
for each row begin 触发器语句…;end;
8. 比较运算符： between ... and 、in 、not in、!= 等
字符串匹配运算符：

判断空运算符:is null 、 is not null 等
位运算符：&、<< select 2 & 3 << 1
子查询运算符：any、some、all
>any(1,2,3)、<all(4,5,6)
9. 系统数据库：information_schema、mysql、sys、performance_schema
information_schema：存储数据库中对象信息（数据库、表、视图、存储过程、触发器等）
mysql：存储 MySQL 数据库正常运行所需要的各种信息（时区、主从、用户、权限等）
sys: 包含了一系列方便 DBA 和开发人员利用 performance_schema 性能数据库进行性能调优和诊断视图
performance_schema: 为 MySQL 服务器运行时状态提供一个底层监控功能，主要用于收集数据库服务器性能参数
10.limit 4,9 的含义：第 5 条到 13 条记录
11.having 作用：去掉不满足条件分组
12.DDL 包括 insert、update、delete 语句
13.delete：彻底删除满足条件或所有记录
14. 多表连接：inner join 、left join 、right join
15.asc、desc 含义：order by name
16.mysql 常识题：
1）端口：3306
2）select 语句至少包含 select from

  1. 删除列：alter table drop

  2. char：固定长度，varchar：可变长度
    5）设置主键自增：id int primary key auto_increment

  3. 子查询：嵌入到另一 SQL 语句中的 select 语句
    7）mysql8 默认的存储引擎是 innodb

  4. sql 是指 structured query language
    9）utf8 编码，一个汉字占用 3 个字节
    10）mysql 语句默认的结束符是；
    11）用户表与部门表之间关系：一个用户对应一个部门；一个部门有多个用户
